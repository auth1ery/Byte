Byte Programming Language - Complete Documentation. Version 1.7

Introduction

Byte is a lightweight interpreted language for simple scripting, rapid prototyping, and educational use. It supports variables, functions, loops, conditionals, buttons, and basic graphics interactions. Future updates may change syntax, so older scripts may require updates.

Core Syntax Rules

  • Scripts must start with STR.
  • Functions and blocks end with end or - (for loops).
  • Declare variables with VAR [name].
  • Commands are case-sensitive.
  • Comments start with // or inline =//.
  • Whitespace is mostly ignored but recommended for readability.

Data Types

  • Numbers: Integers/decimals with + - * /
  • Strings: Wrapped in quotes, concatenated with +
  • Booleans: true or false
  • Objects: Used with Spawn.New, SIPHON, directions

Variables are global unless inside a function. Memory is automatic except const which is immutable.

Control Flow

  • Conditionals: IF / AND / NOT / THN / END
  • Loops: -c + N = ... -
  • Functions: func or onStr

Advanced Functions

  • Change: Modify a script line dynamically.
  • SIPHON: Temporarily siphons objects for N seconds. SIPHON + 5
  • Share: Access data outside current scope. Share LIN 3
  • dleNum / addNum: Increment or decrement numbers quickly.
  • do: Call function or evaluate variable reference.

Performance Tips

  • Keep loops short and avoid deep nesting.
  • Use WIT for delays instead of busy loops.
  • Reuse functions, prefer local variables where possible.
  • Limit heavy use of SIPHON and Share inside loops.

FAQ & Common Issues

  • Unknown command: Check spelling and capitalization.
  • Variable not defined: Declare with VAR first.
  • Infinite loop: Ensure -c + N uses a finite N.
  • Buttons not working: Ensure onStr initializes them and linked function exists.
  • How do I make loops wait? Use WIT = seconds inside the loop.
  • How do I stop a loop? Use STO conditionally.
  • How do I create a button? Button + "Label" EXE [Function] inside onStr.
  • How do I detect/move objects? Use raycast for detection, assign AT / TA / BT / TS for movement.

Best Practices

  • Always start with STR and close loops/functions properly.
  • Use descriptive names and comments for readability.
  • Keep scripts modular, test functions individually, avoid deep nesting.
  • Use const for fixed values and STO to halt safely.
  • Regularly refactor code to remove redundancy and follow latest syntax.
